# Simulation Families

This directory stores generated simulation inputs in two parallel trees:

- `configs/`: scene configs consumed by the simulator and benchmark runners
- `assets/`: per-scene metadata, schedules, and generated audio assets used by those configs

Most benchmark paths expect the config and asset roots for a family to have the same suffix.

## Main Families

### `testing_specific_angles`
Deterministic two-speaker angle sweep used as the base oracle/localization benchmark family. Useful for sanity checks, steering validation, and controlled DOA comparisons.

### `testing_specific_angles_1m_xvf3800`
Angle-sweep scenes targeted at the XVF3800 profile with speakers placed around a roughly 1 m regime. Used for more realistic beamforming/localization comparisons on that array geometry.

### `testing_specific_angles_1m_xvf3800_dry_quick`
Small dry quick-check variant of the 1 m XVF3800 setup. Intended for fast sanity runs rather than realistic reverberant evaluation.

### `testing_specific_angles_2m_xvf3800`
Angle-sweep scenes targeted at the XVF3800 profile with a farther source regime around 2 m. Useful for testing array behavior as distance increases.

### `testing_specific_angles_reappearing_abab`
Turn-taking and reappearing speaker pattern scenes. Useful for tracking/localization persistence checks when speakers leave and return.

### `testing_specific_angles_silence_gaps`
Two-speaker scenes with explicit speech gaps and continuous background noise. Used to test target-activity-gated `Rnn` updates and silence-aware covariance refresh.

### `testing_specific_angles_babble_bootstrap`
Heavy-babble benchmark family with a `4 s` initial noise-only bootstrap. Includes many weak far-field chatter sources plus a WHAM bed. Used to tune detected target activity and bootstrap `Rnn` calibration.

### `testing_specific_angles_near_target_far_diffuse`
Near-target / far-diffuse background family with a `4 s` initial noise-only bootstrap. The target is close to the array while chatter and background are farther away. This is the best current approximation of the intended “foreground speaker in a noisy room” use case.

### `trivial_beamforming_sanity`
Minimal sanity scenes for checking basic steering and beamformer behavior without the full benchmark complexity.

### `trivial_doa_ladder`
Simple DOA ladder scenes with easy angle progressions. Useful for debugging localization behavior and plotting obvious directional changes.

## Other Config Families

### `library_scene`
Library-style meeting/environment scene configs used for broader simulation experiments.

### `restaurant_scene`
Restaurant-style scene configs used for noisier conversational environment experiments.

## Metadata Conventions

Many families write extra files under `assets/<family>/<scene>/`, commonly including:

- `scenario_metadata.json`: scene-level description, timing, DOAs, and family-specific metadata
- `frame_ground_truth.csv`: frame-level target activity / silence / DOA truth when available
- generated source or reference WAVs used by benchmark scripts

The newer bootstrap-oriented families also include explicit metadata for:

- `bootstrap_noise_only_window_sec`
- silence or target-activity schedules
- background noise/chatter layout summaries
- per-frame target activity flags

## Regeneration

These families are generated by scripts in the parent `simulation/` package, for example:

- `python -m simulation.create_testing_specific_angles_scene`
- `python -m simulation.create_testing_specific_angles_silence_gaps`
- `python -m simulation.create_testing_specific_angles_babble_bootstrap`
- `python -m simulation.create_testing_specific_angles_near_target_far_diffuse`
- `python -m simulation.create_trivial_doa_ladder`

Use the matching generator when updating a family so `configs/` and `assets/` stay in sync.
