Title: Add sparse LCMV runtime tuning and expose real-data fast-path timing breakdown

Summary
- Added sparse-bin `lcmv_target_band` solving with complex interpolation and target renormalization.
- Added higher-level runtime controls for LCMV reuse:
  - sparse solve stride
  - sparse solve minimum frequency
  - weight smoothing/reuse
  - DOA refresh tolerance
- Exposed real-data benchmark timing breakdown so `summary_by_method.csv` now carries:
  - `beamforming_rtf`
  - `postfilter_rtf`
  - `pipeline_rtf`
  - `fast_srp_avg_ms`
  - `fast_beamform_avg_ms`
  - `fast_safety_avg_ms`
  - `fast_sink_avg_ms`
  - `fast_enqueue_avg_ms`
  - per-stage fast-path RTFs derived from those averages

Why
- Low-risk tuning alone only moved synthetic `beamforming_rtf` from about `2.22` to `1.50`.
- The dominant cost was still full per-refresh, per-bin LCMV solving.
- Sparse solving was the first architectural change that materially reduced beamforming cost.
- Real-data CSVs were hiding the beamforming-vs-localization-vs-postfilter split, which made RTF comparisons misleading.

New knobs
- `beamformer_sparse_solve_enabled`
- `beamformer_sparse_solve_stride`
- `beamformer_sparse_solve_min_freq_hz`
- `beamformer_sparse_solve_interp`
- `beamformer_weight_reuse_enabled`
- `beamformer_weight_smoothing_alpha`
- `beamformer_doa_refresh_tolerance_deg`

Chosen starting point
- `mvdr_hop_ms = 500`
- `robust_target_band_max_freq_hz = 4000`
- `beamformer_sparse_solve_enabled = true`
- `beamformer_sparse_solve_stride = 4`

Follow-up: DAS attenuation experiments
- Added cheap causal DAS attenuation modes:
  - `delay_sum_subtractive`
  - `delay_sum_subtractive_multi`
  - `delay_sum_differential`
- `delay_sum_subtractive` uses:
  - `target_beam - alpha * interferer_beam`
- `delay_sum_subtractive_multi` uses:
  - target beam minus the average of 3 interferer-steered DAS beams around the cancel angle
- `delay_sum_differential` uses:
  - `target_beam - interferer_beam`
- Added `inverse_rnnoise.wav` export so the removed residual can be inspected directly.
- The inverse RNNoise export is filtered by default to make it less speech-dominant:
  - speech-band stop `300-3400 Hz`
  - `500 Hz` notch

Follow-up: Capon hard-gate tuning and real hop decimation
- Made `localization_hop_ms` a real decimation control:
  - localization now updates every `localization_hop_ms / fast_frame_ms` frames
  - cached localization is reused between updates
- Added Capon runtime knobs across realtime and benchmark paths:
  - frequency-bin subsampling
  - Cholesky/solve path
  - covariance EMA
  - full-scan decimation + local refine
  - grid-size / smoothing / association controls
- Added separate hard-gate tuners:
  - `beamforming/benchmark/tune_capon_realdata_hardgate.py`
  - `realtime_pipeline/tune_capon_synthetic_hardgate.py`

Important bugfixes
- Capon localization backend now actually forwards `grid_size` into `localization/algo.py`.
- Single-source Capon now mirrors `localization_grid_size` instead of silently using the old `72`-bin single-source default.
- Synthetic framewise runs mirror that same single-source grid fix.
