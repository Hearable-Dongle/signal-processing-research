## Summary

Add scene annotation to Data Collection exports, use recording metadata as benchmark ground truth when present, and align the XVF3800 frontend/backend DOA convention.

## What Changed

- extended the Data Collection UI so each recording can carry notes plus a speaker list with names and DOAs, captured before recording and editable afterward
- exported those annotations into each recording's `metadata.json` and surfaced the same summary in `collection.json`
- updated `beamforming.benchmark.data_collection_benchmark` to prefer manual `metadata.json` speakers as ground truth for timeline overlays and static DOA metrics
- changed the XVF3800 simulated/backend geometry to the actual square corner layout and removed the frontend-only `-45°` display compensation
- removed user-facing live `channel_map` overrides so the UI uses a single XVF3800 convention

## Why

This makes real recordings usable for algorithm tuning with manual scene labels, and it removes the DOA mismatch caused by a backend XVF3800 geometry that did not match the frontend visualization.
