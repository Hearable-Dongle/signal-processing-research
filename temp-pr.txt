Simulated beamforming benchmark workflow and UI beamformer-weight surfacing.

What changed:
- Added `beamforming.benchmark.export_simulation_to_collection` to render simulation scene configs into a Data Collection-style export with `collection.json` and per-recording `raw/channel_XXX.wav` files.
- Extended `beamforming.benchmark.data_collection_benchmark` with:
  - `--workers` for parallel recording/method execution
  - simulated-scene ground-truth overlays in `speaker_directions.png`
  - distinct colors for each GT speaker
  - GT drawn above predicted speaker traces
- Added the new testing-angle simulation scene generator and eval entrypoints:
  - `simulation/create_testing_specific_angles_scene.py`
  - `simulation/run_testing_specific_angles_eval.py`
- Updated `simulation/README.md` with the new commands.
- Extended the realtime demo frontend contract to carry beamforming metadata.
- Updated the realtime page and speaker stage to keep the latest beamforming snapshot in UI state.
- Expanded the Beamforming Weights panel to show per-microphone rows with magnitude, phase, and delay when available.
- Added UI test coverage for the microphone-weight rows.
