PR summary

This branch moves the real-data beamforming path toward robust single-target LCMV and adds the debugging hooks needed to inspect each stage on offline recordings.

Main changes
- Added `lcmv_target_band` for single-active beamforming with a protected DOA band around the target.
- Added real-data robustness knobs:
  - trace-scaled diagonal loading
  - identity blending
  - covariance smoothing
  - target-band width
  - target-band high-frequency cutoff
  - target-band condition limit
- Added stage exports and stage PSDs for offline runs:
  - `post_beamforming.wav`
  - `post_wiener.wav`
  - `post_rnnoise.wav`
  - `post_bandpass.wav`
  - `psd_stages.png` plus per-stage PSD plots
- Added postfilter chaining:
  - `wiener_then_rnnoise`
  - `rnnoise_then_voice_bandpass`
  - `wiener_then_voice_bandpass`
- Fixed a zero-output bug in `wiener_then_voice_bandpass` by correctly initializing the bandpass stage.
- Changed the high-frequency LCMV fallback from hard center-only LCMV to target-steered DS above the cutoff.

Why
- Plain single-constraint MVDR was too brittle on real data and often attenuated the target.
- `lcmv_target_band` preserved target energy much better on GT real-data checks.
- On GT `recording-bcs84ogl`, LCMV showed a strong artifact around `15.2 kHz`.
- That was not array aliasing. For the `65 mm` XVF3800 geometry, aliasing is about `2.64 kHz`.
- The actual issue was ill-conditioning:
  - `cond(C^H C) ~= 1.5e8`
- Adding cutoff / conditioning guards removed the spike. The DS-style high-frequency fallback preserves more energy than the first hard fallback.

Key result
- GT `recording-bcs84ogl`:
  - raw RMS: `0.0449`
  - plain `mvdr_fd` post-beamforming RMS: `0.0235`
  - plain `lcmv_target_band` post-beamforming RMS: `0.0436`
- So LCMV preserved target energy much better than MVDR on that GT real-data check.
