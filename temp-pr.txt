Add offline benchmark for Data Collection recordings

What changed:
- Added `beamforming.benchmark.data_collection_benchmark` to run the realtime beamforming stack over Data Collection raw-channel exports.
- Added `realtime_pipeline.recording_runner` so offline recordings can reuse the realtime pipeline and emit `enhanced_fast_path.wav`, traces, metrics, and DOA visualizations.
- Fixed the benchmark module entrypoint so `python -m beamforming.benchmark.data_collection_benchmark ...` actually runs.
- Added explicit pipeline shutdown/join checks so worker threads are stopped before the process exits.
- Documented exact commands for running the utility on an extracted dataset directory or a single recording in `beamforming/README.md`.

How to run:
- Whole extracted dataset:
  `python3 -m beamforming.benchmark.data_collection_benchmark --input-path ../data-collection/fifth-floor-sun-mar-9/ --out-dir ../data-collection/fifth-floor-sun-mar-9/mvd_fr_output --methods mvdr_fd`
- Single recording:
  `python3 -m beamforming.benchmark.data_collection_benchmark --input-path ../data-collection/fifth-floor-sun-mar-9/recordings/recording-08s1khx4/ --out-dir ../data-collection/fifth-floor-sun-mar-9/mvd_fr_output --methods mvdr_fd`

Validation:
- `python3 -m py_compile realtime_pipeline/recording_runner.py beamforming/benchmark/data_collection_benchmark.py`
